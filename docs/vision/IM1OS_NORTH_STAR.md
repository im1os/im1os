# IM1OS North Star

IM1OS exists to give independent powersports businesses the technology, intelligence, and buying power traditionally available only to large dealer groups.

The software is only the beginning.

IM1OS is building an ecosystem where every participating shop becomes more profitable, efficient, and competitive because it belongs to the network.

## Product Identity

IM1OS is not a traditional DMS.

IM1OS is not only shop management software.

IM1OS is an Operating System and Commerce Network for the independent powersports industry.

The first gateway is service and parts operations, but the long-term product is the operating network.

## Product Structure

The long-term product is split into three layers of ownership.

### IM1 Platform

IM1 Platform is IM1's SaaS control center.

It manages:

- Tenant Manager
- Billing
- Merchant onboarding
- Feature flags
- Licensing
- Monitoring
- Deployments
- Support
- Platform analytics

This is above every tenant. It is not part of any dealer's business.

### IM1OS

IM1OS is the tenant operating system used by each shop.

It manages:

- Business administration
- Customers
- Vehicles
- Work orders
- Parts
- Inventory
- Intelligence engines
- Customer portal

### IM1 Network

IM1 Network is the ecosystem layer.

It manages:

- Dealer marketplace
- Shared intelligence
- Rewards
- Procurement network
- Supplier promotions
- Benchmarking
- Social and Market Intelligence
- Future consortium capabilities

The Platform runs the SaaS. The OS runs the shop. The Network connects everyone.

## Mission

Empowering independent powersports businesses through shared intelligence.

## Flywheel

Every repair completed improves estimate accuracy.

Every purchase improves buying intelligence.

Every inventory transaction improves forecasting.

Every supplier promotion improves purchasing recommendations.

Every completed payment improves financial intelligence.

Every participating dealer strengthens the network.

The flywheel:

```text
Service and parts operations
  -> Structured operational data
  -> Better intelligence
  -> Better purchasing and profitability
  -> Stronger supplier participation
  -> Stronger commerce network
  -> More participating shops
  -> Better shared intelligence
```

### Layer 3: Commerce

Scope:

### Layer 4: AI

AI consumes the data and recommendations produced by the lower layers.

AI does not own the data.

AI does not replace the domain model.

AI should explain, summarize, assist, and automate where the operational data, intelligence engines, and permissions allow it.

## Success Definition

Success is not measured only by the number of software subscriptions.

Success is measured by how much more profitable, efficient, and competitive every participating shop becomes because they use IM1OS.

Every new feature should answer one question:

```text
Does this help independent powersports businesses compete more effectively?
```

If the answer is yes, it belongs in IM1OS.

If the answer is no, it probably belongs somewhere else.

## Execution Discipline

The vision is broad, but the build must be phased.

Phase 1: Build the best service and parts operating system for independent shops.

Phase 2: Add the intelligence engines that help shops make better decisions.

Phase 3: Layer in commerce, including merchant services, dealer storefronts, and marketplace capabilities.

Phase 4: Grow the network effects, including benchmarking, shared intelligence, optional inventory exchange, rewards, and supplier partnerships.

Each phase must create value on its own while laying the foundation for the next.
